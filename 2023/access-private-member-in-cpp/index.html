访问类私有成员 | 劫持类私有成员函数 | Folly 库

发表于 2023-09-20 更新于 2025-08-05

最近遇到一个问题, 如何不初始化地对一个 std::vector 进行 resize? resize 会进行初始化, 而 reserve 可以扩容但不会改变 size (实际实现为 begin & end), 因此就可以先 reserve 申请空间之后进行 size 的修改, 但是 size 相关的成员变量其实是 protected.

这就变成了, 如何修改 C++ 类的保护成员? 本文将通过以下几种方法介绍几种私有成员的外部访问方法.

实验在 2Socket * 鲲鹏 920-Server (一共 128C 256G 4NUMA) 进行, 操作系统 CentOS Linux release 7.9.2009 (AltArch), 内核版本 4.14.0-115.el7a.0.1.aarch64, GCC 7.5.0, 开启了 Page Size (65536, 64KB), GNU libc 2.17

1 vector 中 resize 的初始化问题

1.1 为什么想要不初始化?

对于 std::vector, 使用初始化函数和 resize 都会有初始化操作. 但是, 在某些情况下, 我们可能不想.

之前在 博客 里面提到, 对于一个数组, 我们可能想不让他初始化, 在工作线程 First Touch 之后再去分配具体的内存位置, 在某些情况下对性能更友好. 或者有些时候单纯只是不想调用 memset.

2 访问私有成员

不难发现 vector 的基类里面一个 impl 类存放了 begin 和 end. 其中 _M_start 是内存起始地址, _M_finish 是用户可以使用的存储的终止地址, _M_finish - _M_start 就是 size(), _M_end_of_torage 则是内存的末尾地址, _M_end_of_torage - _M_start 就是 capacity().

而 vector 是 protected 继承的

我们可以 reserve() 调整 vector 的 capacity, 之后手动调整 _M_finish.

因此问题就是, 如何修改 vector 的 protected 成员.

2.1 友元

最简单原生的方法就是使用 friend 来访问, 直接类中添加一个 friend 类或者 friend 函数.

但是这样需要修改类定义, 具有侵入性. 比如有些时候想要修改其他库, 则会变得不可维护.
