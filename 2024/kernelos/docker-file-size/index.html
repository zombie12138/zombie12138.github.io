问题

最开始的症状是链接的时候显示网络名不可用:

以往的解决方式是直接重启就好了, 今天想探究一下原因, 服务是 Docker 里面的, 查看一下 Docker 的日志, 还蛮健康

看日志:

似乎是 disk 满了

排查

看一下空间使用

查看 /var 的空间, 初步判断问题是由 Docker 引起的:

进一步查看 /var/lib/docker/, 发现 overlay2 和某个 container 都很大:

这是我写的一个 telegram 的机器人, 出问题的时候会直接 print 堆栈, 但是后来它的代理挂了, 我也没有处理就一直打印网络错误的日志:

就是这个机器人再用:

此外 docker system df (-v) 也可以查看 docker 容器和镜像占据的空间 (-v 是 verbose):

但是 docker system df 命令主要关注 Docker 资源的使用情况, 如镜像, 容器, 卷和构建缓存. 它并不会显示 Docker 引擎内部文件系统的详细信息, 如 overlay2 文件夹或容器的日志文件大小. overlay2 是 Docker 使用的存储驱动之一, 它涉及底层的文件系统实现, 这些通常由 Docker 引擎内部管理

清理空间

首先是可以清理 docker 的无用的空间

上面命令会清理

  • 未使用的容器: 已停止的容器.
  • 未使用的网络: 未被任何容器使用的网络.
  • 未使用的镜像: 没有任何容器依赖的镜像.
  • 构建缓存: 用于 Docker 镜像构建的缓存.

但是仅仅清理了一部分空间, 大概 300 MB. 可以进一步执行以下命令来释放更多空间:

  • -a/--all：除了清理上述未使用的资源外，还会清理所有未被容器使用的镜像，包括那些已经标记 (但没有被使用) 的镜像。如果不使用此选项，只有那些没有任何容器依赖的镜像会被删除。
  • --volumes: 清理所有未使用的卷。未使用的卷是指那些没有被任何容器挂载的卷。这些卷可能会占用大量磁盘空间，因此这个选项可以帮助你释放这些空间。
