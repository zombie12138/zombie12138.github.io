强符号, 弱符号 -- 语言中的各种强和弱

发表于 2024-02-17

最近用GCC@13.2.0去编译Graph500, 发生了报错符号重定义. 结果竟然是…

问题

执行命令:

但是是什么原因呢, 这个就是符号重定义, 打开vscode, 搜索.h或者.c中这个符号(weights)的定义, 发现其定义float *weights;同时存在与csr_reference.c和sssp_reference.c, column同时出现在三个地方.

一开始就大概就猜到是什么原因了

有两种解决办法, 最简单的就是使用extern把除了一个之外的其他所有的定义改成声明, 顺利编译通过. 这个用法的弊端是, 如果这个仅仅声明的文件被单独使用, 会找不到这个符号.

还有一种办法就是在除了一个之外的其他所有的定义或者全部定义同时使用__attribute((weak)), 这样就能顺利编译通过(可能会产生Warning)

Strong Symbol, Week Symbol

但是一个符号, 在多个文件中定义为全局变量, 这个怎么会可以在之前的版本通过编译的. 这个就是未初始化的全局变量本身是弱符号.

这个概念是在<程序员的自我修养>这一本书, P92中看到的, 对于一段C/C++, 未初始化的全局符号就是弱符号

  1. 多个强符号会报错
  2. 多个弱符号会选择空间最大的一个
  3. 如果只有一个是强符号则选择强符号

但是测试的时候发现, 除了attribute手动指定, 似乎都是强符号.

man nm可以看到, T是text代码段, D是initialized data section, B是BSS data section, V是The symbol is a weak object.

此外, 翻gcc手册发现一个标志, 但是似乎没什么作用:

Strong Reference, Weak Reference

除此之外, 还提到了一个概念是强引用, 弱引用.

这个一个就是解决循环引用, 不参与引用计数的引用, 这个在各个语言中都有体现, Python, Java, Cpp也有Weakptr也类似

还有就是这里要讲的, 也是自我修养这本书讲到的.

对于一般的函数声明, 函数定义不在链接中出现, 会报错的

只有链接了add函数定义才可以.

但是如果使用weakref, 就可以在没有定义的时候让函数指针为NULL.

但是测试出问题了… 即使__attribute__((weakref)) int add(int a, int b);也不可以.

查了手册, 想用弱引用就要(必须static), 同时指出引用的函数

如果引用(add_impl)不存在函数入口(add)就是0, 也就是注释掉add_impl就不会执行加法, 这就是如何实现自动适配多线程版本, 如果没有链接多线程的库, 自动不使用

此外有人讲到可以用弱符号解决这个问题(函数声明必须public, 即non static)

w表示 The symbol is a weak symbol that has not been specifically tagged as a weak object symbol. 所以, 还是弱符号

这两个主要是容易混淆, strong, weak指的是类型转换. 而我们容易误以为是动态, 静态类型
