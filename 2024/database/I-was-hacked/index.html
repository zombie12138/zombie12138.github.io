I was hacked

发表于2024-11-19|更新于2025-12-23|Database
|总字数:892|阅读时长:3分钟|浏览量:

我的电脑中病毒了! explorer 也干了!

起因

自从重装了电脑之后, 我的电脑会突然 explorer 内存狂飙, 有的时候 nslookup 内存狂飙. 起初我以为 Windows 的 Bug, 或者我哪里配置有问题, 亦或是 VPN 的原因, 想要排查一下. 于是我打开资源管理器来检查句柄, 网络等资源是否存在异常.

某天突然风扇转起来, 打开资源管理器, 看到两个网络连接, 其中一个带着 miner 相关的字眼, 瞬间气的脑壳疼:

打开两个 URL, 一个是一个 矿池的网页, 另一个是这个攻击者的控制面板:

打开火绒查了一遍病毒, 然后重新启动, 总算消停了.

打垃圾数据

反查和信息收集

先 nmap 扫一遍端口

查到以下端口

  • 21: FTP
  • 22: ssh
  • 80: HTTP
  • 443: HTTPS
  • 888: HTTP

80 端口就是之前的面板, 打开 888 端口提示 403.

于是直接网络上查询这个 IP, 查到和我一样的受害者, 存在一个攻击的 分析报告, 里面记录了被攻击者的网络请求:

这就是我要找的, 打开这个域名, 发现是另一个 panel:

这个就是一个 Github 开源的 panel, 项目名字叫做 UnamWebPanel

下载 db 文件

没想到这么顺利, 由于这个 panel 存了一个 db 文件, 就尝试下载这个网站上的该路径下的 db 文件, 竟然下载成功, 应该是配置没生效. 打开一看, 主要是挖矿的各种配置, 有 27 台 4090, 所有的机器有 4w 台以上:

此外还有一个 config 表, 每次我的电脑挖矿之前都会在这里读取一下配置, 比如使用哪一个矿池, 伪装的可执行程序, 占用的资源等.

Panel 代码分析

如果删除 config, 或者改成错误的配置, 可能就没法害人了（确信

但是看了一下, 几乎每个文件都 include 了密码检查, 只有 endpoint.php 中不需要登陆检查, 但是逻辑比较简单, 没有什么显而易见的漏洞.

但是里面会把当前数据 insert 到 sqlite 数据库中, 虽然没有注入漏洞, 可以给他的数据库塞满没用的数据:

就干脆构造请求来写爆他的 SQLite

反击尝试

直接构造请求, 写入 SQLite, 这样写满之后, 起码不会有新的机器收到矿池配置.

在累计写入一个小时之后, 服务器 SQLite 大小达到了 20 GB, 这个时候会显示, 再查询 12 行的下面的 SQL 的时候会报错.

但当天晚上, 攻击者就删除了数据库并进行了重建, 重新配置了矿池这些信息, 后面准备挂个服务, 只要他服务重置了就继续写.
