灾后重建 -- 从 1 开始安装环境

发表于 2024-04-09 更新于 2024-10-06

有几台几年没关机的 ARM 机器, 一直没人用, 我在上面放了很多程序和实验数据. 结果重装系统了, 并且没有备份, 从 1 开始安装环境.

起因

有 4 台鲲鹏 920 的机器, 每个都有 2 颗鲲鹏 920, 每台 128 核心. 这个机器一直工程师管理 (并没有管理), 很多年没有关机, 用的是 hw 提供的 Centos7.X, Kernel 是 4.x, Kernel 版本低到难用, gcc 是 4.x 的.

昨天天某师弟刚装好环境, 然后就突然告诉我环境没了, 因为他装的不在 home, 放到一个其他硬盘上了, 而我的数据都在 home 目录.
还以为只是格式化了这个硬盘没有波及 home, 正乐呢, 发现我的也没了…
问了一下格式化了, 甚至 home 也没有备份, 上面还有我要用的数据…

好在… 升级了 Kernel, 换了 ubuntu22, 5.15.0-94-generic.

之后找管理员安装 perf, 管理员直接 sudo 用户的密码给我了 (也太草率了). 也就直接给自己开了个用户来装…

同步输入

首先就是生成密钥, 然后配置 config 和 authorized_keys, 四个机器都可以 ssh.

tmux 开了个窗口, 开四个窗口, 分别连四个机器, 然后 set synchronize-panes

安装

为了方便… 直接在 pcie00 上创建了一个目录, 家目录删掉, 软连接到家目录
