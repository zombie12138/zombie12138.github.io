AMD IPU初体验

发表于 2024-02-10 更新于 2024-04-09

买了个火影众颜 众颜U6, 里面自带的IPU(Intelligent Process Unit)据说有5TFLOPS的算力, 但是应该是INT8这种噱头, 但是还是不错的.

但是苦于AMD的IPU除了驱动的支持, 也需要再BIOS中启用, 很多的机器都没有启用, 还专门开了一个issue

最近终于更新BIOS, 设备管理器见到了心心念念的IPU, 可以拿来跑个Hello world了

安装Driver和Ryzen AI Software

除了BIOS, 需要的就是驱动啦, 只要看官方的指南下载安装就可以了

但是这个什么software还需要CMake, 打开vs的命令提示符还需要Conda

我真的很不喜欢这个东西, 开始看这个install.bat到底在做什么, 比Sh还反人类, 手动安装一下, 除了环境检查, 输出提示和错误处理, 就是这几个

看了一下要装的包都是3.9的, 但是… 我是Python3.12, 它只有3.9的包… 研究了一下这个voe主要是Vitis里面的一个东西, 这个vitis似乎是类似类似tensorrt的一个东西.

Vitis AI Execution Provider (Vitis AI EP) 提供了含 AMD DPU 的硬件加速 AI 推断
Vitis AI ONNXRuntime Engine (VOE) 充当的是 Vitis AI EP 的实现库。

找不到任何相关的版本, 只能灰溜溜安装Conda, 然后发现这玩意只能vs2019, 检查的注册表, 按说差不多, 把这部分注释掉就好啦

最后提示环境创建成功就可以了

Hello World

下载下来是一个非常小非常小的鸟图.

环境克隆

为什么要复制一个环境呢… 这是因为… AMD之后的环境坏掉了…

我只装了一些常见的库就不行了(torch套件, jupyter这些), 提示是custom_op.cpp:259: Error: Failed to create GE handle: Error: No compatible xclbin file found.. 懒得弄, 看起来像是VOE的锅.

总结

AMD真的… 开源就生态不行, 文档都找不到, 要么直接不学好不开源. 这个VOE我都不知道在哪找(虽然也没怎么找). IPU更是很多机器没有BIOS开启. 包括文档里面, 一会儿叫IPU一会儿叫NPU…而且很多都是Bin, 教程只有hello world还结构清晰了, 其他都是乱起八糟的…

模型是量化过的, log显示99.5%以上的负载在IPU, 这个模型实测速度和CPU差不多, 看了一下onnx的自带的profile结果, 总感觉哪里怪怪的… 唯一好处好处不太占用CPU资源
