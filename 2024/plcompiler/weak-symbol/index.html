强符号, 弱符号 -- 语言中的各种强和弱

发表于2024-02-17|更新于2025-12-23|PL&Compiler
|总字数:1.4k|阅读时长:5分钟|浏览量:

最近用 GCC@13.2.0 去编译 Graph500, 发生了报错符号重定义. 结果是编译器太新了.

问题

执行命令:

这个就是符号重定义, 打开 vscode, 搜索 .h 或者 .c 中这个符号 (weights) 的定义, 发现其定义 float *weights; 同时存在与 csr_reference.c 和 sssp_reference.c; column 也同时出现在三个地方.

一开始就大概就猜到是什么原因了, 有两种解决办法, 最简单的就是 extern 把除了一个之外的其他所有的定义改成声明, 顺利编译通过. 这个用法的弊端是, 如果这个仅仅声明的文件被单独使用, 会找不到这个符号. 还有一种办法就是在除了一个之外的其他所有的定义或者全部定义同时使用 __attribute((weak)), 这样就能顺利编译通过 (可能会产生 Warning)

Strong Symbol, Weak Symbol

但是一个符号, 在多个文件中定义为全局变量, 这个怎么会可以在之前的版本通过编译的? 原因是之前版本, 未初始化的全局变量本身是弱符号.
这个概念是在 <程序员的自我修养> 这一本书, P92 中看到的, 对于一段 C/C++, 未初始化的全局符号就是弱符号 (书中这样说, 但是高版本只有手动指定才可以)

  1. 多个强符号会报错
  2. 多个弱符号会选择占用空间最大的一个
  3. 如果只有一个是强符号则选择强符号

但是测试的时候发现, 除了 attribute 手动指定, 似乎都是强符号.

man nm 可以看到, T 是text代码段, D 是 initialized data section, B 是 BSS data section, V 是 The symbol is a weak object.

这个还会在模板中广泛使用, 配合 COMDAT 段来避免模板多个实例化的定义错误. 可以参见 链接器这篇文章, 里面 MOLD 链接器的部分提到了模板和弱符号是如何实现的.

2025/7/26 勘误

Gemini 2.5P 说这个 无初始化的全局变量 是 公共符号 (common symbol), 但是表现和弱符号几乎相同. 早期 C 对齐 FORFORTRAN, 但是现代的 C compiler 其实是 -fno-common, 关闭这个特性, 如果开启 -fcommon, 也可以解决这个问题.

也就是说, <程序员的自我修养> 这本书真正的错误是混淆了 公共符号 和 弱符号 的区别, 并且没有说明版本限制.

Strong Reference, Weak Reference

除此之外, 还提到了一个概念是强引用, 弱引用.
这个则是为了实现程序中的可选功能 或者 库中可覆盖的默认实现.

对于一般的函数声明, 函数定义不在链接中出现, 会报错的

只有链接了 add 函数定义才可以. 但是如果使用 weakref, 就可以在没有定义的时候让函数指针为 NULL.

但是测试出问题了… 即使 __attribute__((weakref)) int add(int a, int b); 也不可以.
查了 gcc 手册, 想用弱引用就要 (必须 static), 同时 alias 指出引用的函数

如果引用 (add_impl) 不存在函数入口 (add) 就是 0, 也就是注释掉 add_impl 就不会执行加法, 这就是如何实现自动适配多线程版本, 如果没有链接多线程的库, 自动不使用.
链接之后这个符号就不见了. 但是如果生成 obj 文件, 可以看到

man 查看了下, w 表示 The symbol is a weak symbol that has not been specifically tagged as a weak object symbol. 所以, 也是弱符号. 只是链接的过程中, 删除了这个没有找到定义的符号.

此外 有人 讲到可以用弱符号解决这个问题(函数声明必须 public, 即 non static)

编译链接出来也是弱符号

Strong/Weak Type, Static/Dynamic Type

这两个主要是容易混淆, strong, weak 指的是类型转换. 而我们容易误以为是动态, 静态类型

总结

此外还有动态, 静态 绑定, 这种容易混淆的概念好多. 也发现 自我修养 这本书里错误挺多的.
