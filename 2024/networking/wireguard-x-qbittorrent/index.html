在 Wireguard 里面跑 Qbittorrent

发表于2024-04-24|更新于2025-12-23|Networking
|总字数:1.2k|阅读时长:5分钟|浏览量:

之前研究过 Qbittorrent 怎么利用公网 IP, 单纯设置自己上报的 IP, 同时反代自己监听的端口似乎还不够, 还是直接 Wireguard 里面装一个 Docker 比较合适.

正好最近的 OneCloud 一台小机器放家里, 做跳板机, 配置了 DDNS, 有公网 IP, 而且带宽比腾讯云的大不知道哪里去了

Server - Wireguard + iptable

Wireguard 会自动把 client 的转发过来, 自动配置 Nat, 但是似乎不太会端口转发, 也不会自动暴露端口, 这个都需要我们去手动的配置. 有个 dyonr/qbittorrentvpn 的 Docker, 但是由于一开始没有使用, 就一直没用, 而且 qbit 的版本肯定也会受限. 一致依赖我都是自己用 Wireguard 和 Qbittorrent 拼出来的. 下面介绍下我的方案.

首先就是在 Docker 启动的时候要执行一段 iptable, 设置端口转发, 8999 的 tcp 和 udp, 以及 8888 的 tcp.

我这里要每次执行 Docker 之前都自动配置 iptable, 可以在 wireguard 的基础上生成 Docker 镜像, 也可以直接修改 Docker 入口. 我们选择使用 docker-compose 来修改 docker 的入口, 用下面的这个脚本来作为入口, 记得 chmod +x init-iptables.sh

这个 wireguard 的 Docker 原来的入口就是 ENTRYPOINT ["/init"], 这里就直接用下面的 docker compose 文件

配置了一个 Peer, 就是客户端的 Wireguard, 他会负责转发 Qbittorrent 的流量, 默认监听的是 51820 的 UDP 端口, 这里改成了 54, 并且暴露 54; 8999 端口则作为 Qbittorrent 和 peer 沟通的端口; 8888 端口作为 web 控制台.

按说第一次使用这个 Docker, 会在配置文件里面产生自己的公钥私钥, 可以查看 cat /apps/wireguard/config/peer1/peer1.conf, 这个就是 peer 需要的
