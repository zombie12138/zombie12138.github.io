AMD IPU 初体验

发表于2024-02-10|更新于2025-12-23|Architecture
|总字数:800|阅读时长:3分钟|浏览量:

买了个火影众颜 U6, 里面自带的 IPU (Intelligent Process Unit) 据说有 5 TFLOPS 的算力, 但是应该是 INT8 这种. 但是苦于 AMD 的 IPU 除了驱动的支持, 也需要 BIOS 中启用, 很多的机器都没有启用, 还专门开了一个 issue

最近终于更新 BIOS, 设备管理器见到了心心念念的 IPU, 可以拿来跑个 Hello world 了

安装 Driver 和 Ryzen AI Software

除了 BIOS, 需要的就是驱动, 只要看官方的 指南 下载安装就可以了

但是这个什么 software 还需要 CMake, 需要 vs, 还需要 conda

我真的很不喜欢 conda, 开始看这个 install.bat 到底在做什么, 准备尝试手动安装一下, 除了环境检查, 输出提示和错误处理, 就是这几个

看了一下要装的包都是 3.9 的, 但是… 我是 Python3.12, 它只有 3.9 的包… 研究了一下这个 voe 主要是 AMD 的 Vitis 里面的一个东西, 这个 vitis 似乎是类似类似 NV 的 TensorRT 的一个东西.

Vitis AI Execution Provider (Vitis AI EP) 提供了含 AMD DPU 的硬件加速 AI 推断
Vitis AI ONNXRuntime Engine (VOE) 充当的是 Vitis AI EP 的实现库。

找不到任何相关的版本, 只能灰溜溜安装 conda, 然后发现这玩意安装时候会通过注册表检查是否安装 vs2019, 但是我安装的 vs 是更高的版本, 只能手动注释这部分检查.

最后提示环境创建成功就可以了

Hello World

下载下来是一个非常小非常小的鸟图.

环境克隆

为什么要复制一个环境呢? 这是因为这个环境莫名其妙会坏掉. 我只装了一些常见的库就不行了 (torch 套件, jupyter 这些), 提示是 custom_op.cpp:259: Error: Failed to create GE handle: Error: No compatible xclbin file found.. 懒得弄, 看起来像是 VOE 的锅.

总结

AMD 虽然开源, 但是生态依然很差. 文档都找不到, 甚至有些组件直接不开源, 只给 Bin. 这个 VOE 我都不知道在哪找 (虽然也没怎么找). IPU 更是很多机器没有 BIOS 开启. 甚至官方介绍的文档里面, 一会儿叫 IPU 一会儿叫 NPU, 教程只有 hello world 结构清晰, 其他都是乱七八糟的.

模型是量化过的, log 显示 99.5% 以上的负载在 IPU, 这个模型实测速度和 CPU 差不多, 可能和模型太小有关. 看了一下 onnx 的自带的 profile 结果, 总感觉哪里怪怪的, 准备后面再仔细折腾.
