TVM 入门 -- 0 Hello World

发表于 2024-05-03 更新于 2025-08-05

来系统看一下 TVM 的教程, 本文将包含安装和一些基本的使用, 基本上参考 照抄 tvm 官方文档 和 中文文档

简介

TVM 是一个深度学习编译器

其步骤就是:

  1. 从 TF, Torch, 或者 ONNX 的模型, 导入
  2. 把模型翻译成 Relay IR, 会加入一些图优化
  3. 降级为 TE (Tensor Expression), FuseOps pass 会把模型切分为小图.
  4. 使用 auto-tuning 模块 AutoTVM 或 AutoScheduler 搜索最佳 schedule 进行一些循环优化 (切分, 融合, 展开等), 并行化之类的优化
  5. 生成 JSON 的调优记录, 为每个子图选择 schedule
  6. 降级为 TensorIR, 进一步优化 (降级到硬件)
  7. 生成机器码

Install

跟着教程就行了

注意 Cmake 要 3.20 以上. 我开启了 CUDA, NCCL, NVTX, OpenCL, OpenMP, RELAY_DEBUG 和 LLVM, 一些库没有就可以在 CMakeList.txt 里面添加一些 link_directories 或者 include_directories 来解决

最终效果就是 python 里面能 import tvm, 并且能执行 python -m tvm.driver.tvmc

比较坑的是 llvm 版本, 官网只说了 4.0 以上, 但是 17 不可以, 下面的时候会 Segment Fault, 14 就可以… 搜了一下, 官网没有进行其他额外的说明, 而且也没有搜到什么要求.

Tune

在 AMD EPYC 7742 上表现如下, TVM 会不断尝试不同的参数, 进行 PGO, 记录最佳成绩. 每个 Task (可能是一个 kernel) 执行了 40 轮的调试:

这里搜索 Schedule (Step 4): 对于上一步生成的每一个子图 (Kernel), TVM 进行自动调优, 它会为每个Kernel尝试成百上千种不同的实现方式 (Schedule). 会执行例如:

  • 循环分块 (Tiling): 将大循环切成小块以利用缓存
  • 循环展开 (Unrolling): 减少循环判断开销, 也可以更好利用 SIMD 等优化.
  • 向量化 (Vectorization): 使用 CPU 的 SIMD 指令并行处理数据。

tvmc tune 会在真实硬件上运行这些不同的Schedule, 并测量其性能 (GFLOPS), 记录当前性能和最佳性能. 当所有任务完成后，tvmc tune 会将每个 Kernel 的最佳 Schedule记录下来, 保存到输出文件 resnet50-v2-7-autotuner_records.json 中.

输出文件里面是不同次运行的, 主要包含输入输出的分块, 循环展开的配置和其测量得到的性能.
