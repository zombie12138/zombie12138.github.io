此外docker df ls (-v)也可以查看docker容器和镜像占据的空间 (-v是verbose):

但是docker df命令主要关注Docker资源的使用情况, 如镜像, 容器, 卷和构建缓存. 它并不会显示Docker引擎内部文件系统的详细信息, 如overlay2文件夹或容器的日志文件大小. overlay2是Docker使用的存储驱动之一, 它涉及底层的文件系统实现, 这些通常由Docker引擎内部管理

清理空间

首先是可以清理docker的无用的空间

上面命令会清理

  • 未使用的容器: 已停止的容器.
  • 未使用的网络: 未被任何容器使用的网络.
  • 未使用的镜像: 没有任何容器依赖的镜像.
  • 构建缓存: 用于Docker镜像构建的缓存.

但是仅仅清理了一部分空间, 大概300M

可以进一步执行下面的来

  • -a/--all：除了清理上述未使用的资源外，还会清理所有未被容器使用的镜像，包括那些已经标记（但没有被使用）的镜像。如果不使用此选项，只有那些没有任何容器依赖的镜像会被删除。
  • --volumes: 清理所有未使用的卷。未使用的卷是指那些没有被任何容器挂载的卷。这些卷可能会占用大量磁盘空间，因此这个选项可以帮助你释放这些空间。
