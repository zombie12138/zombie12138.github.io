计算机相关的英文的专有名词, 缩写和名言警句

发表于2024-05-04|更新于2025-12-23|Collections
|浏览量:

一些计算机相关的名词, 缩写和名言警句的收录. 这些术语看似高深, 实则阐述的都是些常见的概念.

Terminology

  • Early Return: 一般使用卫语句 (Guard Code) 来在入口检查. 例如 if (p) { return; }
  • Materialization Model: 物化模型, 每个 operator 一次处理所有的输入, 处理完之后将所有结果一次性输出, 更适合 OLAP 负载
  • Volcano Model / Iterator Model / Pipeline Mode: 火山模型是数据库查询优化和执行中最为常用的执行模型. 每一种操作抽象为一个 Operator, 整个 SQL 查询被构建成一个 Operator 树。 查询执行时，树自顶向下调用 next() 接口, 数据则自底向上被拉取处理, 因此这种处理方式也被称为拉取执行模型 (Pull Based)
  • Vectorized / Batch Model: 向量化模型和火山模型 类似，每个operator需要实现一个 next() 函数，但是每次调用 next() 函数会返回一批的元组，而不是一个元组，所以向量化模型也可称为批处理模型
  • Merkle Tree: 哈希组成的树

Abbreviation

  • RAII: Resource Acquisition Is Initialization.
  • JIT: Just-in-time (compilation)
  • AOT: Ahead-of-time (compilation)
  • OLAP: Online analytical processing
  • OLTP: Online Transaction Processing
  • ETL: extract, transform, and load
  • NFR: NonFunctional Requirement
  • DevOps: Development & Operations
  • CI/CD: Continuous Integration / Continuous Delivery
  • QA: Quality Assurance
  • UAT: User Acceptance Testing
  • MESI: Modify Exclusive Shared Invalid
  • SATB: Snapshoot At The Beginning
  • CMF: Concurrent Mode Failure
  • TLAB: Thread Local Allocation Buffer
  • RTTI: Run-Time Type Identification
  • FOSS: Free & Open Source Software
  • OSR: On Stack Replacement
  • ARM: Automatic Resource Management
  • DCE: Dead Code Elimination
  • SSA: Static Single Assignment
  • DDD: Domain Driven Design
  • JEP: Java Enhancement Proposal
  • PDM: Performance Diagnostic Model

Theory / Law

  • Little’s law: 系统中客户的长期平均数量 L 等于长期平均有效率 λ 乘以客户在系统中花费的平均时间 W
  • Amdahl’s law: 优化系统的单个部分获得的整体性能改善, 受到改进部分用时占比的限制
  • Wadler’s law: Wadler’s law 是 Law of Triviality 在编程语言设计领域的一个著名实例. 当设计一种语言时，花在语言结构上的时间与这些特征的重要性相比很不成比例。

Aphorisms

OPTIMIZATION

Premature optimization is the root of all evil
过早的优化是万恶之源
             —Donald Knuth

Data dominates. If you’ve chosen the right data structures and organized things well, the algorithms will almost always be self-evident. Data structures, not algorithms, are central to programming.
数据占主导地位. 如果您选择了正确的数据结构并很好地组织了事物, 那么算法几乎总是不言而喻的. 编程的核心是数据结构,而不是算法
           —Rob Pike’s 5 Rules of Programming

Profile

Don’t guess, but measure!
不要猜测, 而是测量

COMMON

Done is better than perfect
完成比完美更重要
