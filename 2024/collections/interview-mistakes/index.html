面试中的误解

发表于2024-04-25|更新于2025-12-23|Collections
|总字数:502|阅读时长:1分钟|浏览量:

总结以下面试中遇到的误解.

算法

文远某行 - 树的直径

求树的直径. 这个不算错误, 应该是误解

我不刷 Leetcode 的, 我还以为, 树的直径是指 以每个节点为根进行层次遍历, 找到节点数最多的那一层的节点数, 最后在所有情况中取最大值..

这个感觉可以记忆化搜索或者其他什么的, 但是时间复杂度都是 O(n^2) 什么的. 但是面试官跟我说的是可以记忆化搜索, 也可以 DP, 但是时间复杂度 O(n), 直接傻眼了, 面试官很无奈说可能太难了, 最后做了个链表, 草草把我挂了…

某天突然还是感觉不可思议, 怎么可能 O(n)的, so 了一下, 原来树的直径是树中最长的路径…

这件事告诉我们: 一定要重新表述和面试官重新沟通自己对于题目含义和自己思路的理解

操作系统

某公司 - SO 中共享变量

主管面, 问我如何在 so 中确保数据共享.

首先就是所有的跨进程的同步的手段都可以使用, 各种锁. 最关键的是我问 so 中可以直接共享变量吗, 面试官说可以的, 并且明确表明是一个 so 库中的全局变量. 但是事实是 linux 中的 so 库的全局的区域是每个进程独一份, 没有共享的.

某米 - BSS 和段

面试官问我局部变量存哪, 讲了半天我说了一句未初始化 static 的局部变量在可执行文件的 bss 段, 程序加载时, 这部分数据会被映射到进程地址空间的数据段中, 也就是和全局数据区一起, 面试官没听直接问: “局部变量存在全局数据区吗”, 我又重新解释了一遍, 他还是表示怀疑…
