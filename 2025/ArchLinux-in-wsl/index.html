ArchLinux & Windows 开发环境配置

发表于 2025-08-31 更新于 2025-11-03

I use Arch Linux btw.

一直以来都是用的 Debian 系的, 也用过 Redhat 和 CentOS, 很少用过一些其他的发行版, 比如Gentoo, ArchLinux, openSUSE. 虽然这些虽然是外面配了一些不同的软件, 但是还是感觉非常神秘. 最近重装系统了, 顺便在 wsl 中安装 ArchLinux 体验一下. 之前也从来没有做过什么终端美化, 这次简单尝试下.

首先是安装 WSL2, 直接 follow 微软的 官方教程 就可以了, 重启后在 Store 中找到 ArchLinux, 点击安装, 打开, 配置用户名和密码就好啦.

oh-my-zsh

首先安装 zsh, 并且设置为默认

然后安装 oh my zsh

之后 sudo su root 重新给 root 也配置下.

之后配置下主题和插件, 主要参考 Leehow 和 arch gaidline.

oh my zsh 主题众多, 可以 ls ~/.oh-my-zsh/themes 看到全部的主题. 参考一个主题的 投票排名, 最终选择最热门的 Powerlevel10k. 类似风格的还有 Powerlevel9K, Bullet Train, agnoster.

之后设置 .zshrc 修改变量 ZSH_THEME="powerlevel10k/powerlevel10k". 直接 source 执行可以进入配置, 重新配置可以 p10k configure. 按照提示来选择喜欢的配置即可, 也可以参考 梁臣洋博客.

小玩具

还有一些没用的小玩具. 本来没想安装这么多的, 只是最多安装个 fastfetch 就好了. 但是安装 zsh 的时候看到一篇 博客 感觉很有意思.

但是一个问题就是, 截屏 scrot 返回的是空白的花纹; import 会直接卡住. LLM 给出的原因是:

当您在 Windows 上以 多窗口模式 (multi-window mode) 运行 VcXsrv 或 X410 时, X Server 会创建一个 不可见的、虚拟的桌面背景, 这就是 “根窗口”. 您启动的每一个 Linux GUI 程序 (如 xeyes, gedit)都会在这个根窗口之上, 被渲染成一个独立的, 由 Windows 窗口管理器负责管理的 Windows 窗口. 所以, 当您运行 scrot 时, 它忠实地完成了任务: 连接到 VcXsrv, 截取了整个 “根窗口”. 但这个根窗口本身是空的, 或者只有一个默认的灰度花纹背景.

当您完成拖拽操作后, import 就会截取您选择的区域, 保存为 screenshot.png, 然后命令才会结束并返回到您的终端提示符. 因为您不知道它在等待您的操作, 所以看起来就像是卡住了.

所以看起来还是只能截图 WSL 产生的 GUI, 截图基本上还是没什么用. 还以为获取整个 shell 文字渲染成好看的图片, 原来是直接截图 Linux GUI, 运行时候还在想为什么还需要 X11-Forward.

总结

sudo -E 还是有一些风险的, 只是本地使用, 当时没有给 root 配置 proxy. 可以直接 修改镜像 或者配置 root 代理 / 透明代理即可.

安装环境还是适可而止, 现在这些各种 oh-my, 已经在我家目录生成了大量的隐藏文件夹, 以及 trae, vscode, cursor 的, ls -alh 已经不太人类可读了. 并且好看的工具层出不穷, 是否真正提升生产力还是未知数, 比如 .tmux 感觉就完全没必要安装. 我个人感觉 tmux + vim 基本操作 + vscode remote 体验就已经不错了.
