OnePlus13 一加13 Root

发表于 2025-09-20

最近买了个 OnePlus 13, 简单记录 Root 的过程.

上上次 Root 还是高中 16 年那个能通过漏洞一键 Root 的时代, 上次 Root 则是我的 Redmi Note 12 Turbo, MIUI 14.

警告: 手机 Root 存在一定的风险, 并且不同手机解锁 BL 锁流程不同.

为什么 Root

其实并没有非要 Root 的需求, 主要原因是更喜欢自己买的手机自己掌控的感觉, 安装例如 存储隔离 等的工具, LSPosed 的大量插件我用的倒不是很多 (唯一一个一直用的就是 Bilibili 禁止竖屏模式). 我当前的设备:

可以 magisk patch 下 kernelsu patch 产出的 init_boot.img, 重新刷入; 或者直接授予 Magisk root 权限, 然后点击安装即可实现两者共存. 但是安装 Magisk 会导

应用冻结

其实更多的是应用自动关闭或者冻结, 同类竞品很多, 最终选择了最新的时尚单品 雹.

  • 绿色守护, 可以 root 或者开启无障碍来关闭或者休眠应用. 早期还可以, 但是高版本 Android 不怎么用.
  • 黑域: 通过 ADB/Root 自动 force-stop, 旧系统更合适
  • 冰箱: Root/Shizuku, disable-user 冻结.
  • 雹: Root 等多种方式实现, 可以冻结或者关闭后台应用, disable-user/force-stop.

我对于隐藏 root 其实一直没有那么感兴趣, 奈何有一些应用, 例如银行类的会去检测. 检测 Root 包含可疑 APP, Zygisk, 可疑挂载等. 这里使用 Native Detector 进行检测.

其他

其他的应用都是看自己喜好了, 比如: Fake Location, 应用列表隐藏, Disable-FLAG_SECURE, 传感器滤镜, KernelFlasher 等. 还有一个就是 Youtube, 可以自己 revanced manager 来进行 patch, 我更喜欢直接安装 别人做好的.

总结

后面想装什么插件, root 软件直接安装就可以. 升级系统的话, 可能会覆盖之前的 init_boot.img, 还要重新 patch 刷入或者使用 KernelFlasher, 因此自动更新我是关闭并且冻结的.

Root 依然很方便, 最大的阻碍还是 Root 生态逐渐凋零, 各种组件都停止维护, 例如 LSPosed 官方停止维护, 之前使用的 Bilibili 禁止竖屏也很多年不再更新. 更大的问题是厂家也在尽量阻挠用户获取 Root 权限, 比如小米从等待 7 天, 变成签到打卡之后考试申请, 考试难度不断提升, 最终变成了 体育特招 才能解锁 BL; OnePlus 的 BL 权限也在 不断收紧, 可能过不了几年, 市面上就找不到能解锁 BL 锁的国产手机了.
