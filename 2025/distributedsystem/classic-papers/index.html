那些经典论文

发表于2025-03-08|更新于2025-12-23|DistributedSystem

记录 2012 年之前的又老又经典论文阅读.

MapReduce: Simplified Data Processing on Large Clusters

MapReduce[1] 是一种通用的 Bulk Synchronized Parallel 的典型范式. 用户只需要定义一个 Map 函数, 一个 Reduce 函数即可在上千台实例上运行分布式任务, 可以完成 PageRank, Inverted Index 等任务. 例如词频统计的一个例子:

*注: 当时是 20 年前, 机器还是双核 2-4 GB, 架构, 主频等都很羸弱* 1. Split Input 到 16 MB ~ 64 MB. 2. 大量启动程序, 其中一个是 Master; 其余由 Master 调度, 分为 M 个 Map, R 个 Reduce. 3. Map 读取输入, 解析 K/V, 传给 Map 函数, 生成 KV 对到内存中. 4. 缓存中的 KV 对被分为 R 个区域, 定期刷入到 Map worker 的本地的磁盘上. 位置上报 MASTER. 5. Reduce 收到 Master 发来的文件位置, 通过 RPC 读取, 按 Key **排序** (可能外部排序). 6. 每一个 Key, 传入一个 Reduce 函数, 输出到一个分区的输出文件中. 7. 完成后, Reduce **重命名** 输出文件 (atomic), 上报 Master. 一些调度细节: - Worker 失效: Master 周期性 Ping Worker, 如果没有收集到某个 worker 的消息标记为失效. 任何失效机器上的任务都会标记为 idle 重新调度. 已经完成的 Map 也需要重新执行 (因为存储在本地), 下游没有读取的 Reduce 也会被通知从新的输出读取数据, 而 Reduce 不需要, 因为其输出到全局文件系统 (GFS). - Master 失效: 可以 Master 定期 checkpoints, 但是由于情况较少, 设计较复杂, 因此直接失败. - 存储: 采用 GFS 存储, 每个分块 64 MB, 存放 3 拷贝, 尽量将 Map 调度到输入更近的机器上执行. - 任务粒度: 正常情况下 M & R 会比集群中 Worker 机器多的多. 这样能更好的进行负载均衡, 失效时候也可以更快恢复. 但是 Master 决定了不能太多, master 执行 `O(M+R)` 次调度, 保存 `O(M*R)` 个状态. 比如使用: 2000 workers, M=200000 R=5000 - 长尾: 使用 Backup Tasks, 当 MR 接近完成, master 启用 Backup Tasks 来执行 in-progress tasks. 任何一个完成, 该任务即被标记为完成. 一些实现细节: - Partitioning Function: 用户可以自定义 Partitioning Function, 否则使用 Hash + MOD 来进行分区. - Combiner Function: Map 可能存在大量的重复的记录, 比如之前的词频统计会产生大量 `` 的记录, 在 Reduce 拉取 Map 结果之前先执行自定义 Combiner Function, 可以有效节约网络开销. - Ordering Guarantees: 给定分区, 中间的 kv pair 是有序的. 这对于需要按键进行高效随机查找的场景很有用. - Skipping Bad Records: 有些记录本身可能就损坏, 在一种模式下可以跳过这种记录. 每个 Worker process 配置一个 signal handler 捕捉 segmentation violations 和 bus errors, 如果遇到错误, 处理函数会 UDP 发送给 Master, 当多次在同一个位置的损坏记录失败, 就会下次执行跳过. - Local Execution: 由于分布式执行, 因此 Debug 变得困难, MR 提供了一种本地执行的方式, 在本地串行执行 MR, 也可以指定执行某个 Map. 一个 Sort 工作, 可以看到, backup tasks 可以有效减少尾延迟; 同时细粒度的调度和重试机制可以有效应对大量任务的失败: 相关工作包括 GFS, River 也是经常听到, 可以后续看下. 可能是因为看过类似的介绍, 看完的感受是, 收获并没有想象中大, 原因大概是这种 MapReduce 的思想和技术实现已经大量被后续的系统所借鉴, 并且在一些其他地方也会多少学习到 MapReduce. 这个论文的的地位和含金量毋庸置疑, 而且最后引用也只有 20 个, 不知道是那时候论文的风格还是这个论文太弔了.
  • [1] Dean, Jeffrey, and Sanjay Ghemawat. Mapreduce: simplified data processing on large clusters. osdi. Vol. 4. No. 1. 2004.
